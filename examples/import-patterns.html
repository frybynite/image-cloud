1. Tree-Shaking: Import Single Layouts

‚úÖ Recommended for Production

Import only the layout(s) your app needs:

import { ImageCloud } from '@frybynite/image-cloud';
import '@frybynite/image-cloud/layouts/radial.js';

const gallery = new ImageCloud({
  container: 'gallery',
  images: [...],
  layout: { algorithm: 'radial' }
});
‚úì Benefit: Only radial layout is bundled. Unused layouts (grid, spiral, etc.) are tree-shaken by your bundler. Smaller bundle = faster downloads.

2. Multi-Layout: Import All Layouts

For Apps Using Multiple Layouts

Import all layouts at once:

import { ImageCloud } from '@frybynite/image-cloud';
import '@frybynite/image-cloud/layouts/all.js';

const gallery = new ImageCloud({
  container: 'gallery',
  images: [...],
  layout: { algorithm: 'spiral' } // Any layout works
});
‚úì Benefit: All layouts are available. Useful for dynamic layout switching or multi-gallery apps.

3. Auto-Init: Pre-Bundled with All Layouts

For Script Tag Usage

Use auto-init bundle for CDN/script tag scenarios:

<link rel="stylesheet" href="https://cdn.example.com/image-cloud/style.css">
<script src="https://cdn.example.com/image-cloud/auto-init.js"></script>

<div data-image-cloud data-config='{ "layout": { "algorithm": "radial" }, "images": [...] }'></div>
‚úì Benefit: Self-contained bundle. All layouts included. No imports needed.

4. Framework Wrappers

React, Vue 3, Web Components

// React
import { ImageCloud } from '@frybynite/image-cloud/react';
import '@frybynite/image-cloud/layouts/grid.js';

// Vue 3
import { ImageCloud } from '@frybynite/image-cloud/vue';
import '@frybynite/image-cloud/layouts/grid.js';

// Web Component
import '@frybynite/image-cloud/web-component';
import '@frybynite/image-cloud/layouts/grid.js';
‚úì Benefit: Framework-specific wrappers are ~1KB each (external dependency). Still import layouts separately.

5. Specific Loader

Dynamic Loader Imports (Static Only)

Loaders are dynamically imported. Use static URLs:

import { ImageCloud } from '@frybynite/image-cloud';
import '@frybynite/image-cloud/layouts/radial.js';

const gallery = new ImageCloud({
  container: 'gallery',
  images: [           // Uses StaticImageLoader (imported dynamically)
    'image1.jpg',
    'image2.jpg'
  ]
});
‚úì Benefit: GoogleDriveLoader is NOT bundled if not used. Saves ~8.5KB.

Summary: Bundle Size Comparison

Scenario Bundle Size What's Included
Single Layout Only ~133KB Core + 1 layout
All Layouts ~150KB Core + 6 layouts
Auto-Init Bundle ~110KB Core + all layouts + all loaders
React Wrapper (separate) ~0.8KB React wrapper (core is external)

Helpful Tips

üí° Tree-Shaking Works Best When:
  • You use a modern bundler (Webpack, Vite, Rollup, esbuild)
  • You import only the layouts you need
  • Your bundler has tree-shaking enabled (default in most modern setups)
‚ö†Ô∏è Common Mistakes:
  • Importing @frybynite/image-cloud/layouts/all.js bundles ALL layouts (no tree-shaking)
  • Not importing any layout causes helpful error: "Layout algorithm X is not registered"
  • Using auto-init in production when single-layout import would be smaller
